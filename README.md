## Teamate Project Background
* 필요성
  * 높은 리플레이 가치 확보
  * 절차적 맵 생성 기술을 활용한 맵 리소스 제작 비용 감소로 인한 향후 인디 게임 개발 시 효율적인 솔루션 제공
  * 캐주얼 E스포츠 종목으로의 확장 가능성 

    
* 기존 해결책의 문제점
  * 기술적 문제점(동기화 및 성능)
  * 복잡하고 방대한 맵 생성에서 일어나는 프레임 저하(Stuttering) 등의 과도한 연산 부하 
  * 매번 완전히 새로운 구조의 맵에 오브젝트를 배치하고 컬링(Culling)하는 과정 자체가 고정 맵보다 더 많은 오버헤드를 발생시키는 오브젝트 풀링의 한계
